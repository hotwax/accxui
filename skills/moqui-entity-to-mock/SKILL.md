# Moqui Entity to Mock Data Conversion

This skill provides instructions for converting Moqui entity exports (JSON format) into TypeScript mock data files for frontend application development.

## Context

This practice is suitable when:
- The frontend application fetches entities directly via a CRUD REST API.
- The entity data structure in Moqui matches the expected data structure in the application.
- Simple exports of entities can serve as reliable references for API calls.

> [!IMPORTANT]
> This method is NOT recommended if the API layer performs significant data transformations, aggregations, or if the application requires complex business logic to be simulated in mocks.

## Workflow

1.  **Identify the Export File**: Typically a JSON file generated by Moqui's entity export feature (e.g., `EntityExport_YYYYMMDD_HHMM.json`).
2.  **Parse and Group**:
    - The export is a JSON array of objects.
    - Each object contains a `_entity` field identifying its type.
    - Group objects by their `_entity` type.
3.  **Sanitize Data**:
    - Remove Moqui-specific metadata fields like `_entity` from the final mock objects.
    - Ensure dates and other field types are handled consistently.
4.  **Generate TypeScript Modules**:
    - For each group, create a `.ts` file.
    - Use the naming convention: `mock[EntityName]`.
    - Export a constant array: `export const mock[EntityName] = [...];`.
5.  **Placement**:
    - Place the generated mock files in the application's mock directory (e.g., `src/mock/`).
    - Use the `convert_export.py` script located in this skill's `scripts/` directory for automated conversion.

## Conversion Script Usage

```bash
python3 scripts/convert_export.py --input-file <path_to_export> --output-dir <path_to_mocks> --entities <comma_separated_entities>
```
